    • The structured querying capability ensures that you can retrieve specific information from your data lake without drowning in chaos.

    • BigLake is an extension of BigQuery that enables you to query data stored in supported external data stores. It assumes familiarity with database tables and Identity and Access Management (IAM).
    • You can create BigLake tables for the following data stores:
      • Cloud Storage: Create Cloud Storage BigLake tables and query them using GoogleSQL syntax.
      • Amazon S3: Similarly, create Amazon S3 BigLake tables and query them.
      • Azure Blob Storage: Create Azure Blob Storage BigLake tables and query them.
    • Access Delegation: BigLake tables decouple access to the table from access to the underlying data store. A service account handles data retrieval from the data store, allowing you to enforce fine-grained security at the table level.
    • Cross-Cloud Joins: BigLake enables cross-cloud joins, allowing you to run queries that span both Google Cloud and BigQuery Omni regions. You can analyze data across various storage solutions (AWS, Azure, public datasets, etc.) without copying data between sources.
    • Dynamic Data Masking: For BigLake tables based on Cloud Storage, you can use dynamic data masking.
