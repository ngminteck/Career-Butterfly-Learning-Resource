Windows Template Library (WTL) is a free software, object-oriented C++ template library designed for Win32 development. It was initially created by Microsoft employee Nenad Stefanovic for internal use and later released as an unsupported add-on to Visual Studio and the Win32 Framework SDK1. Here are some key points about WTL:

  1. Purpose and Origin:

    • Objective: WTL provides a lightweight alternative to the Microsoft Foundation Classes (MFC) for creating Windows applications.
    • Origin: Originally developed for internal use, it builds upon Microsoft‚Äôs ATL (Active Template Library), which is widely used for creating COM and ActiveX libraries.

  2. Features and Benefits:

    • Efficiency: WTL aims to deliver small and efficient code, similar in size and speed to ‚Äústraight‚Äù SDK programs.
    • Object Model: It offers a higher-level and more flexible object model to developers.
    • UI Elements: WTL supports various user interface elements, including frame and popup windows, MDI, standard and common controls, common dialogs, property sheets, GDI objects, scrollable windows, splitter windows, toolbars, and command bars.
    • Familiar Interface: Most of the WTL API mirrors standard Win32 calls, making it familiar to Windows programmers.

  3. Licensing:

    • Originally, WTL had a license similar to the one used for the MFC source code but without usage or distribution restrictions.
    • In 2004, Microsoft made the complete source code available under the Common Public License and released it through SourceForge.
    • Since version 9.1, the library is licensed under the Microsoft Public License1.
