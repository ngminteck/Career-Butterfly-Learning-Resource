TestNG, which stands for Next Generation, is a powerful and flexible testing framework for Java applications. It was developed by Cedric Beust and is inspired by both JUnit and NUnit. Here are some key features of TestNG:

  1. Annotations: TestNG introduces annotations that allow you to define test methods, setup methods, and teardown methods. Annotations provide flexibility and control over test execution.

  2. Parallel Execution: You can run your tests in arbitrarily large thread pools with various policies available (such as running each method in its own thread or one thread per test class).

  3. Data-Driven Testing: TestNG supports data-driven testing using the @DataProvider annotation. You can easily parameterize your tests and run them with different input data.

  4. Flexible Test Configuration: TestNG allows you to configure your tests using XML files or programmatically. You can specify test groups, dependencies, and other settings.

  5. Support for Parameters: You can pass parameters to your test methods using the @Parameters annotation.

  6. Powerful Execution Model: Unlike JUnit, TestNG does not rely on a test suite. Instead, it uses a more flexible execution model. You can organize your tests using groups, dependencies, and other attributes.

  7. Integration with Tools and Plugins: TestNG is supported by various tools and plugins, including Eclipse, IntelliJ IDEA, and Maven.

  8. Multithread Safety Testing: You can test whether your code is multithread safe using TestNG.

  9. Embeds BeanShell: TestNG embeds BeanShell, which provides additional flexibility for scripting.

  10. No Need to Extend Classes: Unlike JUnit, there‚Äôs no need to extend a specific class or implement an interface. You can use any method names you like, and TestNG recognizes them based on annotations.

    @BeforeClass
    public void setUp() {
        // Code that will be invoked when this test is instantiated

    }

    @Test(groups = {"slow"})
    public void aSlowTest() {
        System.out.println("Slow test");
    }
}

In this example:

  • The setUp() method is invoked before any test method runs.
  • We have two test methods: aFastTest() and aSlowTest().
  • The @Test(groups = {"fast"}) annotation specifies that aFastTest() belongs to the ‚Äúfast‚Äù group.

You can run your TestNG tests from the command line, using Ant, or via an XML file. For example, with Ant:

<project default="test">
    <path id="cp">
        <pathelement location="lib/testng-testng-5.13.1.jar"/>
        <pathelement location="build"/>
    </path>
    <taskdef name="testng" classpathref="cp" classname="org.testng.TestNGAntTask"/>
    <target name="test">
        <testng classpathref="cp" groups="fast">
            <classfileset dir="build" includes="example1/*.class"/>
        </testng>
    </target>
</project>
