  1. Logic-Based Approach:

    • In Prolog, logic is expressed as relations, which are known as facts and rules.
    • The core concept of Prolog lies in applying logic to these relations.
    • Computation is carried out by querying these relations.

  2. Knowledge Base and Facts:

    • Prolog programs consist of a knowledge base that contains facts.
    • Facts represent entities and their relationships.
    • Each fact ends with a dot (.).
    • For example:
      • friends(raju, mahesh). (Raju and Mahesh are friends)
      • singer(sonu). (Sonu is a singer)
      • odd_number(5). (5 is an odd number)

  3. Unification and Backtracking:

    • Unification is a fundamental concept in Prolog.
    • Prolog checks if given terms can represent the same structure.
    • Backtracking occurs when a task fails, and Prolog attempts to satisfy previous tasks.

  4. Running Queries:

    • You can ask queries in Prolog using the following format:
      • ?- singer(sonu).
        • Output: Yes (since this fact exists in the knowledge base)
      • ?- odd_number(7).
        • Output: No (since this fact is not in the knowledge base)

  5. Advantages:

    • Easy to build databases without extensive programming effort.
    • Pattern matching is straightforward.
    • Built-in list handling simplifies working with algorithms involving lists.
