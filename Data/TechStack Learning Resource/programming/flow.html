Flow-based programming (FBP) is a programming paradigm that views applications as networks of black box processes. These processes, also known as nodes, communicate by exchanging data across predefined connections using message passing. Here are the key aspects of flow-based programming:

  1. Network of Processes:

    • In FBP, an application is represented as a network of interconnected processes.
    • Each process acts as a black box that performs specific tasks or transformations.
    • These processes communicate by passing data between them.

  2. Data Flow Emphasis:

    • Unlike traditional programming, where control flow dictates the execution order, FBP focuses on data flow.
    • The goal is to transform and move data efficiently through the network of processes.
    • Data flows asynchronously between processes in the form of information packets (IPs).

  3. External Connections:

    • The connections between processes are specified externally to the processes themselves.
    • A piece of software, often called the scheduler, interprets the network definition.
    • Fixed-capacity connections link processes, allowing data to flow between them.

  4. Component-Oriented:

    • FBP naturally supports a component-oriented approach.
    • Processes (components) can be reconnected endlessly to create different applications without internal changes.
    • This flexibility enables modular design and easy reuse.

  5. Optimal Resource Utilization:

    • FBP processes continue executing as long as they have data to work on and a place to output results.
    • Applications generally run in less elapsed time than conventional programs.
    • All available processors on a machine are optimally utilized without special programming.

  6. Visual Representation:

    • FBP network definitions are often diagrammatic.
    • The diagram specifies the connections between processes.
    • Some FBP languages/runtimes provide visual programming interfaces.
