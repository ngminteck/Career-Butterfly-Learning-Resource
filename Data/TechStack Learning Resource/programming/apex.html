    • Custom Public APIs: Built from stored Apex methods.

  4. Rigorous and Multitenant Aware:

    • Strongly Typed: Direct references to schema objects (e.g., object and field names) ensure compile-time validation.
    • Metadata Dependencies: Custom fields, objects, and classes are stored in metadata to prevent deletion while needed by active Apex code.
    • Multitenant Environment: Like the rest of the Lightning Platform, Apex runs in a shared environment, guarding against runaway code and resource monopolization.
