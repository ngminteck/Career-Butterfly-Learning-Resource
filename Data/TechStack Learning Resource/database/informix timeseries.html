  1. What Is Time Series Data?

    • A time series consists of time-stamped data. These data points are collected at specific intervals or timestamps.
    • Examples of time series data include:
      • Electricity usage recorded by smart meters.
      • Stock prices and trading volumes.
      • ECG (electrocardiogram) recordings.
      • Seismograms.
      • Network performance records.

  2. Capabilities of Informix TimeSeries Solution:

    • Data Structure Definition:
      • Define the structure of the time series data.
    • Data Acceptance Control:
      • Set the frequency for regularly spaced records.
      • Handle arbitrarily spaced records.
    • Data Storage Control:
      • Specify where to store data.
      • Change where data is stored.
      • Monitor storage usage.
    • Data Querying:
      • Extract values for a specific time range.
      • Find null data.
      • Modify data.
      • Display data in standard relational format.
    • Data Analysis:
      • Perform statistical and arithmetic calculations.
      • Aggregate data over time.
      • Make data visible or invisible.
      • Find the intersection or union of data.

  3. Advantages of Informix TimeSeries Solution:

    • Flexible and Customizable:
      • Not specific to any industry.
      • Easily customizable.
    • Combines Relational and Non-Relational Approaches:
      • Can combine time series data with information in relational databases.
    • Efficient Storage and Querying:
      • Saves disk space by avoiding duplicate information.
      • Loads and queries time-stamped data quickly due to optimized storage order.

  4. Comparison with Standard Relational Tables:

    • While relational databases can store time series data by storing one row per time-stamped entry, performance is poor and storage is inefficient.
    • The Informix TimeSeries solution overcomes these limitations by using a specialized format within a relational database, resulting in faster data loading and more efficient storage.

In summary, the Informix TimeSeries solution bridges the gap between non-relational and standard relational implementations, making it a versatile choice for handling time series data in various domains12.
