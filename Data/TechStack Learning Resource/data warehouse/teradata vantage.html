  1. Parsing Engines (PE):

    • When you run a SQL query in Teradata, it first reaches the Parsing Engine.
    • The PE performs several essential functions:
      • Manages individual user sessions (up to 120).
      • Checks if the objects used in the SQL query exist.
      • Verifies if the user has the required privileges for the objects used in the query.
      • Parses and optimizes the SQL queries.
      • Prepares the execution plan and passes it to the corresponding Access Module Processors (AMPs).
      • Receives the response from the AMPs and sends it back to the requesting client.

  2. BYNET:

    • BYNET is a communication system that enables seamless interaction between components.
    • Key functions of BYNET:
      • Coordinates multi-AMP queries.
      • Reads data from multiple AMPs.
      • Regulates message flow to prevent congestion.
      • Enables Massively Parallel Processing (MPP) capabilities.

  3. Parallel Database Extension (PDE):

    • Positioned between the operating system and the Teradata Vantage database.
    • Facilitates features like BYNET and shared disks.
    • Responsible for the parallelism that ensures speed and scalability.

  4. Access Module Processors (AMPs):

    • Responsible for data storage and retrieval.
    • Each AMP is associated with its own set of Virtual Disks (Vdisks).
    • AMPs operate in parallel, adhering to the shared-nothing architecture.
    • Functions of AMPs include accessing storage using Vantageâ€™s Block File System Software and managing locks.

  5. Virtual Disks (Vdisks):

    • Assigned to AMPs in enterprise platforms.
    • In VantageCloud Lake environments, Vdisks belong to the Primary Cluster.
    • Vdisks store data and ensure isolation between AMPs.
